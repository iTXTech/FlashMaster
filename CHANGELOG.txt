Version 2.2.0
- Added a Settings controller group card. FlashMaster now stores the selected controller groups locally and applies them to PN decode, PN search, Flash ID decode, and Flash ID search.
- Improved parser capability details with grouped controller inventory, localized group names/descriptions from capabilities, default group display, and expandable controller lists.
- Added build-time sidebar footer notice variables for ICP filing text or other public deployment information, with optional external links.
- Improved history-mode deployment on Cloudflare Pages with static asset passthrough, SPA route fallback, and a dedicated in-app 404 page that marks unknown routes as noindex.
- Unified vendor logo rendering across PN and Flash ID decode results, including dark-logo handling and more consistent sizing across themes.
- Reduced search-result noise by hiding controller projection fields from compact result summaries and Flash ID result tables.
- Cleaned up the changelog dialog by removing the redundant subtitle while keeping full hash-bearing FlashMaster version display and normalized read-state tracking.
- Embedded parser updates are summarized in the fdnext v2.2.0 release notes: https://github.com/iTXTech/fdnext/releases/tag/v2.2.0

Version 2.1.0
- Added PWA support to the regular web build. Run `pnpm build`, serve `dist` over HTTPS, then install FlashMaster from the browser or home-screen prompt; the cached app shell can reopen after the first successful visit.
- Added `pnpm build:singlefile` for self-contained offline HTML distribution as `FlashMaster-<version>-<commitHash>.html`.
- Updated embedded fdnext to v2.1.0 (6a48b2e), hiding internal decodepack token and grouping fields from public PN decode results.
- Updated embedded fdnext to v2.1.0 (268679e), adopting the renamed decodepack package layout for embedded parser builds.
- Removed the obsolete `deploy.bat` GitHub Pages deployment script.
- Repositioned FlashMaster as a Memory Chip Intelligence Platform in app copy and documentation.
- Reworked the public route set to `/parts`, `/ids`, `/settings`, and `/about`, with optional `/en` and `/zh` language prefixes.
- Added `VITE_FLASHMASTER_ROUTER_MODE=hash|history`; hash mode remains the default, while history mode uses normal URLs with SPA rewrite support.
- Updated embedded fdnext to v2.1.0 (4c1c609), expanding DRAM and managed NAND coverage with CXMT DRAM process decoding, Silicon Motion managed NAND, SK hynix ZUFS 4.1, and broader legacy Micron eMCP decoding.
- Simplified the About project section so it only emphasizes the GNU AGPL 3.0 license instead of repeating the source link and HTTP API mode.
- Updated embedded fdnext to v2.0.0 (fe99a8c), using fdnext standard build metadata with 7-character commit hashes.
- Settings parser/server information now displays fdnext commit hash and build time from the capabilities API for both embedded and HTTP modes.
- Added built-in fdnext HTTP API presets for iTXTech fdnext Cloud and Local fdnext Dev.
- Updated embedded fdnext to v2.0.0 (c0e005d), adding the shared runtime adapter layer and the canonical ExternalLink result contract.
- Added ExternalLink rendering for decode results and search cards, including vendor-logo link images and opt-in preview data injected through an fdnext processor.
- Updated embedded fdnext to v2.0.0 (de51be4), deduplicating part-search marking candidates for cleaner FBGA and package-marking search results.
- Split controller fields into collapsible lists across decode and search result views.
- Refined wide-screen layouts by centering search/settings workspaces, keeping decode details denser, and tightening English UI labels.
- Updated embedded fdnext to v2.0.0 (9b3ec2f), adding the capabilities inventory contract for parser, FDB, resource, and decoder metadata.
- Reworked Settings parser information into structured cards for parser version, FDB metadata, inventory counts, decoder lists, and supported capabilities.
- Updated embedded fdnext to v2.0.0 (36fa208) and migrated all decode/search paths to the fdnext 2.0 canonical result contract.
- Rebuilt PN and Flash ID result rendering around canonical devices, blocks, relations, warnings, candidates, and search items.
- Changed HTTP parser mode to fdnext 2.0 endpoints only and removed external server-list fetching.
- Improved dense wide-screen and narrow-screen layouts for result headers, field tables, relation rows, search cards, and controller summaries.
- Removed the legacy bit-unit preference because fdnext 2.0 now supplies canonical display units per field.
- Updated embedded fdnext to v1.1.0 (7176c5b), adding PN parsing support for DRAM, eMCP, eMMC, UFS, Managed NAND, and more.
- Managed NAND: expanded PN DSL coverage for SK hynix, Samsung, SanDisk/KIOXIA, YMTC, Micron, and vendor metadata.
- DRAM: added Micron/Crucial, SK hynix, Samsung, Nanya, Elpida, and CXMT DRAM PN decoding, resources, docs, and tests.
- DRAM output: refined public type and CE/Die classification output, reducing redundant internal fields in DRAM decode results.
- Micron FBGA/MDB: added unified Micron FBGA resources, refreshed MDB data from the improved crawl, DRAM PN search data, raw MDB refresh flow, and startup fixes around bundled resources.
- FDB: added FirstChip controller generation sources and expanded FlashDB controller plus PN/Flash ID matching data.
- Managed NAND suggestions: added structured PN suggestion resources and unified compat baseline checks.
- Vendor logos: filled logo mappings for fdnext-supported vendors and tuned original logo artwork sizing for Kingston, Elpida, and related brands.
