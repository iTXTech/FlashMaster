Version 2.6.0-dev (Snapshot date: 2026-07-12)

FlashMaster updates:
- Reorganized the Commercial Banner capabilities dialog into application/platform, data/automation, and systems/specialized-engineering groups, with lower-pressure “Discuss a project” and concise “View capabilities” actions.
- Refined the dense workstation themes with separate dark and light neutral ramps, clearer light-mode page, panel, and active-navigation hierarchy, no transient light strip above Market Pulse after switching themes, better-aligned dark input surfaces, a balanced Settings side rail, and focused PN/Flash ID decode-and-search actions, while preserving the established panel borders and compact Market Pulse size.
- Stabilized Market Pulse on mobile Safari: quote sources now merge into a stable asset set while preserving the current visible ticker, visible slots reuse their DOM nodes, and CSS-driven scrolling continues through independent cycles instead of relying on an infinite-animation boundary and negative-delay resume. This prevents brief full-row ticker swaps and auto-scroll stalls while retaining horizontal drag behavior.
- Rebuilt PN and Flash ID result layouts around responsive auto-flow cards: every detail card keeps a single-column footprint so long controller and relation lists grow vertically without creating cross-column holes, wide screens backfill uneven card heights, narrow screens collapse cleanly to one column, and dynamic expansions remeasure in place. Very long controller lists now expand into a bounded multi-column scroller instead of stretching indefinitely or disappearing behind the bottom banner. Flash ID External Links now join the same flow and fill the next available slot instead of leaving an empty column.
- Reduced the PWA precache by keeping the embedded fdnext worker offline-ready while loading the duplicate main-thread fallback engine only on demand when online.
- Hardened embedded parser failover: a timed-out or broken worker is now terminated and disabled so the current and subsequent operations fall back without repeated 30-second waits.

fdnext v3.2.0 highlights:
- Integrated the latest fdnext v3.2.0 snapshot (62a3133) in the embedded parser.
- Expanded Flash ID coverage for Macronix parallel NAND, Winbond, and legacy NAND/MCP identifiers, and added broader GDDR7 and CXMT decoding.
- Expanded PN and catalog coverage for SK hynix, Micron eMMC, Winbond, Samsung, KIOXIA, YMTC, and other managed NAND and DRAM families.
- Separated DecodePack evidence metadata from runtime rules to keep source references auditable without changing FlashMaster's parser interface.
- Unified modern SanDisk iNAND eMMC/UFS token decoding while retaining legacy part-number coverage.
- Expanded PN decoding across Biwin, ESMT, Kingston, KIOXIA, Longsys, Micron, Samsung, SK hynix, SpecTek, Winbond, YMTC, and other priority vendors, including broader DRAM, HBM, Raw NAND, SPI NAND, eMMC, eMCP, UFS, and SSD structures.
- Expanded managed NAND and DRAM search resources and refined vendor identifier profiles for broader lookup coverage.
- Fixed YMTC Gen5 Flash ID post-processing so matching identifiers resolve to the correct die profiles.
- Expanded Managed NAND decoding for ISSI, Biwin, Kingston, Longsys, Samsung, SanDisk, and Micron parts, including new eMMC, eMCP, UFS, and SSD structures.
- Expanded DRAM decoding and search resources for ISSI, ESMT, Nanya, Samsung, and Micron, while deduplicating Micron PN resources.
- Corrected Micron raw NAND DecodePack feature tokens: `Q` now reports Enterprise Q, and `QZ` / `ZQ` report Enterprise Q + Polyimide Process Applied.
- Tightened Micron raw NAND optional-suffix parsing around feature and production-status tokens at colon/end-of-input boundaries.
- Reworked the embedded runtime around prepared catalogs, indexed FDB lookups, prefix-dispatched DecodePack matching, and projected search decoding to reduce repeated initialization and per-query work.
- Compacted eager search indexes to reduce retained runtime data while preserving exact, prefix, and substring search behavior.
- Optimized embedded search paths for complete prefix and substring matching, while keeping explicit top-K suggestion queries on the faster ranked path.
- Cleaned public DRAM output by removing internal configuration/package codes and unsupported default die-count assumptions, and normalized SK hynix H27 raw NAND package labels.

Version 2.5.0 (Release date: 2026-07-06)
Special thanks to HFB for providing reference material that helped refine the Raw NAND decoding rules in this release.

FlashMaster updates:
- Improved PWA update freshness while preserving offline cache: entry HTML, public SPA routes, service-worker files, and the manifest now revalidate on open, hashed static assets stay long-cached, and installed users refresh once after a new service worker takes control.
- Improved embedded fdnext responsiveness in web and full/nano single-file builds: the parser now warms up after the first paint where appropriate and runs embedded decode/search work asynchronously in a module worker, while regular web builds keep a main-thread fallback and HTTP-only builds remain available.
- Improved PN and Flash ID suggestions by keeping debounce behavior, clearing stale candidates immediately for new queries, and avoiding UI-render blocking during embedded searches.
- Polished the dense workstation UI after the dependency refresh, including sidebar text colors, menus, suggestion rendering, controller-group selection, snackbar placement, button contrast, and relation-card focus/border behavior.
- Improved Market Pulse chart behavior: desktop ticker clicks and horizontal drags now use a cleaner gesture threshold, clicks reliably open charts near the clicked ticker, drags or horizontal scroll close open charts, and ticker auto-scroll resumes after a drag closes an open chart.
- Added a polished FlashMaster display label for fdnext's new `3d_xpoint` chip kind in result badges and capability views.

fdnext v3.1.0 support highlights:
- Integrated fdnext v3.1.0 (d7d9655) in the embedded parser.
- Expanded Raw NAND PN decoding across SK hynix, Samsung, Micron, KIOXIA, SanDisk, SpecTek, and Intel, including broader SK hynix H25/H27/HY27 routing, H2E/H2N details, gated SK hynix 3D die-profile identifier matching, structured Micron MT29FB/HSC handling, improved Samsung stack/die/suffix/package decoding, and cleaner KIOXIA/SanDisk/SpecTek/Intel QLC profile normalization.
- Added Intel and Micron 3D XPoint part-number decoding with the new `3d_xpoint` chip kind.
- Expanded Micron DRAM coverage across DDR3/DDR3L/DDR4/DDR5 packages, `D9XLQ` TwinDie DDR3L, EDY DDR4, JE/KH DDR4 packages, and TwinDie / QuadDie / DDR4 3DS stack-series output.
- Expanded Micron SSD DecodePack coverage for broader client and automotive SSD part-number structures, and improved SpecTek DRAM product-code and mobile DRAM package decoding.
- Added DecodePack array-table support, normalized package outputs, refined MCP public fields, separated DRAM die-count geometry, reduced decode warmup overhead, and tightened public-output wording/code-token audits.
- Updated FDB relation trimming to match PN and Flash ID relations against DecodePack NAND die/profile metadata, producing more accurate Raw NAND aliases around KIOXIA/SanDisk, Micron/SpecTek, Samsung, and SK hynix records.
- Added the FlashDetector-compatible `fd-server` package upstream, including Node.js and Cloudflare Workers deployment support for legacy HTTP routes.
- Full parser and resource details are tracked in the fdnext release notes: [iTXTech/fdnext](https://github.com/iTXTech/fdnext/releases/tag/v3.1.0)

FlashMaster updates:
- Added PWA support to the regular web build. Run `pnpm build`, serve `dist` over HTTPS, then install FlashMaster from the browser or home-screen prompt; the cached app shell can reopen after the first successful visit.
- Added `pnpm build:singlefile` for self-contained offline HTML distribution as `FlashMaster-<version>-<commitHash>.html`.
- Added tag-triggered GitHub Actions release packaging for single-file, hash-router, history-router, and checksum artifacts.
- Removed the obsolete `deploy.bat` GitHub Pages deployment script.
- Repositioned FlashMaster as a Memory Chip Intelligence Platform in app copy and documentation.
- Reworked the public route set to `/parts`, `/ids`, `/settings`, and `/about`, with optional `/en` and `/zh` language prefixes.
- Added `VITE_FLASHMASTER_ROUTER_MODE=hash|history`; hash mode remains the default, while history mode uses normal URLs with SPA rewrite support.
- Simplified the About project section so it only emphasizes the GNU AGPL 3.0 license instead of repeating the source link and HTTP API mode.
- Settings parser/server information now displays fdnext commit hash and build time from the capabilities API for both embedded and HTTP modes.
- Added built-in fdnext HTTP API presets for iTXTech fdnext Cloud and Local fdnext Dev.
- Added ExternalLink rendering for decode results and search cards, including vendor-logo link images and opt-in preview data injected through an fdnext processor.
- Split controller fields into collapsible lists across decode and search result views.
- Refined wide-screen layouts by centering search/settings workspaces, keeping decode details denser, and tightening English UI labels.
- Reworked Settings parser information into structured cards for parser version, FDB metadata, inventory counts, decoder lists, and supported capabilities.
- Rebuilt PN and Flash ID result rendering around canonical devices, blocks, relations, warnings, candidates, and search items.

- Improved dense wide-screen and narrow-screen layouts for result headers, field tables, relation rows, search cards, and controller summaries.
