Version 2.4.0 (Release date: 2026-05-31)
FlashMaster updates:
- Added the HTTP-only `pico` single-file flavor. It removes embedded fdnext from the bundle, inherits the nano feature trimming, shows `HTTP only` in parser version surfaces, and can lock the HTTP server at build time with `VITE_FLASHMASTER_LOCKED_SERVER`.
- Improved PN and Flash ID query entry with a shared suggestion input, cleaner formatted input handling, and consistent decode/search behavior across direct entry, suggestions, language switching, and controller selections.
- Reworked decode and search result presentation with shared detail blocks, expandable nested values, clearer relation labels, fewer repeated query echoes, and denser result tables.
- Updated Settings parser information for fdnext v3 capability data, including compact backend-owned inventory metrics, controller-group exclusivity, clearer HTTP-only/pico version surfaces, and fresh capability requests instead of stale cached data.
- Polished workstation UI behavior with unified scrollbars, tighter mobile/PWA overlay spacing, steadier bottom panels, and compact browser title/metadata handling for active PN and Flash ID subjects.
- Improved optional Market Pulse presentation with smoother visible-slot scrolling, quieter desktop chart placement, and less distracting behavior while hidden or off-screen.
- Added vendor logo coverage for ESMT, Etron, GigaDevice, ISSI, Silicon Motion, and Winbond decodepack vendors, and switched GigaDevice to official SVG artwork for sharper result and link rendering.

fdnext v3.0.0 support highlights:
- Integrated fdnext v3.0.0 (1f7cbfc) in the embedded parser.
- Expanded DRAM PN coverage across Samsung, SK hynix, Nanya, ISSI, Winbond, ESMT, Etron, CXMT, and GigaDevice.
- Expanded managed NAND, MCP, UFS, and eMCP coverage, including SK hynix H25/UFS/eMCP/uMCP, Samsung MCP, Micron managed/eMCP/FBGA, SpecTek suffixes, and KIOXIA raw/managed/UFS rules.
- Improved NAND Flash ID decoding with Samsung QLC die profiles, Samsung geometry decoding, and updated SK hynix, SanDisk, and KIOXIA identifier profiles.
- Added Phison `T27HGA5A1V`, refreshed FDB V84 data, and expanded controller/PN/Flash ID coverage including YS9205 and SM2268XT2 mappings.
- Reduced duplicate managed NAND fields and kept internal implementation fields out of public results.
- Full parser and resource details are tracked in the fdnext release notes: [iTXTech/fdnext](https://github.com/iTXTech/fdnext/releases/tag/v3.0.0)

Version 2.3.0 (Release date: 2026-05-17)
FlashMaster updates:
- Active PN and Flash ID pages now rerun the current decode/search automatically after language switching, keeping visible results in the selected language without duplicating local usage counts.
- Added optional Market Pulse enhancements, including a quiet Settings toggle, visible-slot rendering, price-change flashes, and click-through candlestick/volume charts backed by cached Hyperliquid market snapshots.
- Reworked Settings parser information into native capability cards for parser build metadata, FDB metadata, inventory counts, controller groups, supported operations, and decoder lists.
- Hardened controller-group selection against stale or invalid capability data, falling back to reported defaults while preventing empty selections.
- Added the `pnpm build:singlefile:nano` offline build flavor for a smaller single-file artifact with analytics and Market Pulse disabled.
- Reduced regular and nano single-file bundle size by manually registering only the Vuetify components used by the UI, without adding a Vuetify Vite plugin.
- Polished app chrome by keeping PN/Flash ID subjects in the browser title and SEO metadata while leaving the in-page title compact, plus improved mobile overlay spacing for bottom panels.

fdnext v2.3.0 support highlights:
- Integrated fdnext v2.3.0 (b0010c0) in the embedded parser.
- Added shared NAND die profile metadata and moved NAND Flash ID density/codename logic into DecodePack identifier rules.
- Expanded coverage for Samsung DRAM suffixes, DDR3 naming tokens, Micron DRAM/NAND colon revision tokens, Micron 2100AI/2100AT SSDs, and YMTC raw NAND/process metadata.
- Updated bundled FDB to V83, added `fdnext.fdb.v1`, cleaned cross-vendor and low-confidence records, and normalized process fields to die-profile keys.
- Added fdfdb extra overlays and audit-extra tooling so candidate PN/Flash ID data can be reviewed before entering generated FDB output.
- Full parser and resource details are tracked in the fdnext release notes: [iTXTech/fdnext](https://github.com/iTXTech/fdnext/releases/tag/v2.3.0)

- Improved parser capability details with grouped controller inventory, localized group names/descriptions from capabilities, default group display, and expandable controller lists.

- Reduced search-result noise by hiding controller projection fields from compact result summaries and Flash ID result tables.
- Cleaned up the changelog dialog by removing the redundant subtitle while keeping full hash-bearing FlashMaster version display and normalized read-state tracking.

fdnext v2.2.0 support highlights:

- Expanded DecodePack coverage across NAND, Managed NAND, DRAM, HBM/HMC, SpecTek, KIOXIA, Samsung, Micron, ISSI, ESMT, Etron, and Phison rules.

- Added controller group resources and capability filtering for more precise controller projection.

Version 2.1.0 (Release date: 2026-05-11)
FlashMaster updates:
- Added PWA support to the regular web build. Run `pnpm build`, serve `dist` over HTTPS, then install FlashMaster from the browser or home-screen prompt; the cached app shell can reopen after the first successful visit.
- Added `pnpm build:singlefile` for self-contained offline HTML distribution as `FlashMaster-<version>-<commitHash>.html`.
- Added tag-triggered GitHub Actions release packaging for single-file, hash-router, history-router, and checksum artifacts.
- Removed the obsolete `deploy.bat` GitHub Pages deployment script.
- Repositioned FlashMaster as a Memory Chip Intelligence Platform in app copy and documentation.
- Reworked the public route set to `/parts`, `/ids`, `/settings`, and `/about`, with optional `/en` and `/zh` language prefixes.
- Added `VITE_FLASHMASTER_ROUTER_MODE=hash|history`; hash mode remains the default, while history mode uses normal URLs with SPA rewrite support.
- Simplified the About project section so it only emphasizes the GNU AGPL 3.0 license instead of repeating the source link and HTTP API mode.
- Settings parser/server information now displays fdnext commit hash and build time from the capabilities API for both embedded and HTTP modes.
- Added built-in fdnext HTTP API presets for iTXTech fdnext Cloud and Local fdnext Dev.
- Added ExternalLink rendering for decode results and search cards, including vendor-logo link images and opt-in preview data injected through an fdnext processor.
- Split controller fields into collapsible lists across decode and search result views.
- Refined wide-screen layouts by centering search/settings workspaces, keeping decode details denser, and tightening English UI labels.
- Reworked Settings parser information into structured cards for parser version, FDB metadata, inventory counts, decoder lists, and supported capabilities.
- Rebuilt PN and Flash ID result rendering around canonical devices, blocks, relations, warnings, candidates, and search items.
- Changed HTTP parser mode to fdnext 2.0 endpoints only and removed external server-list fetching.
- Improved dense wide-screen and narrow-screen layouts for result headers, field tables, relation rows, search cards, and controller summaries.
- Removed the legacy bit-unit preference because fdnext 2.0 now supplies canonical display units per field.

fdnext v2.1.0 / v2.0.0 support highlights:
- Integrated fdnext v2.1.0-era embedded parser updates (6a48b2e, based on v2.1.0) and the fdnext 2.0 standard result contract.
- Adopted the DecodePack package layout, shared runtime adapter layer, canonical ExternalLink result contract, and fdnext build metadata with 7-character commit hashes.
- Expanded DRAM and managed NAND coverage, including CXMT DRAM process decoding, Silicon Motion managed NAND, SK hynix ZUFS 4.1, and broader legacy Micron eMCP decoding.
- Added capabilities inventory for parser, FDB, resources, registered decoders, commit hash, and build time.
- Migrated decode/search rendering to canonical devices, blocks, relations, warnings, candidates, search items, typed Flash ID identifiers, and deduplicated marking candidates.
- Full parser and resource details are tracked in the fdnext release notes: [v2.1.0](https://github.com/iTXTech/fdnext/releases/tag/v2.1.0), [v2.0.0](https://github.com/iTXTech/fdnext/releases/tag/v2.0.0)
